# Research Findings Report — GCD-as-Authorization for LLM Agents

**Date:** 2026-06-22 · **Status:** Results from three completed runs (`Host A` 2× RTX 5090 + `Host B` 4× RTX 5090).
**Scope of this report:** what the three finished datasets actually show, mapped to the pre-registered RQs/DVs, with limitations stated up front. This is a findings memo, not the paper.

**Data sources (gitignored result DBs):**
- `harness/blast_1p7b_5m.db` — the 1.7B (Qwen) generator×gate factorial, **2,370,741 trials**, 0 errors.
- `harness/mistral119_anchor.db` — the **capable non-Qwen anchor** (Mistral-Small-4-119B-NVFP4, 4×5090 TP=4), **37,872 trials**, 0 errors.
- `harness/steelman_abliterated.db` — the abliterated-adversary steelman (Qwen3.5-27B), **4,000 trials** (2,000/cell), 0 errors.

---

## 0. One-paragraph summary

Three datasets across **two pretraining families (Qwen + Mistral)**. **(1)** On a deliberately **weak workhorse (Qwen3-1.7B-NVFP4)**, a generator×gate factorial at ~282k attack trials/cell reproduces **constitutive-vs-corrective** decisively (RQ2/RQ3/RQ6): GCD arms emit **zero** out-of-scope artifacts and make the allowlist gate **inert**, while behavioral generators behind the same gate produce 104k–120k out-of-scope calls each (produce-then-catch) at ~2× the tokens. **(2)** On a **capable non-Qwen anchor (Mistral-Small-4-119B-NVFP4, 4×5090 TP=4, n=4,000/cond)** the matrix runs the behavioral arms **ungated**, delivering what the weak victim could not: **RQ1** — the full stack A4 still leaks **1.82% [1.43, 2.29]** (CI-lower > 0), A1 (56.50%) ≤ Control (56.77%), Control clears the ≥50% floor — and the **formal H2 contrast**, C **0.00% [0, 0.09]** strictly below every behavioral point estimate. It also shows the blind-C deflection cost is a **weak-model artifact**: blind-C legit-success **78.85% > Control 73.56%** (point estimate; the 1.7B's 11-pp deflection deficit does not reproduce — but this is a point-estimate ordering, **not** the registered powered 2pp non-inferiority TOST, which is not powered at n=208/`--legit-rounds 1`), with a large **validity dividend** (C 99.8% valid vs ~59% for native free-generation). **(3)** A separate **abliterated-adversary steelman** (Qwen3.5-27B, malicious top-authority prompt) shows a **97.85% → 0%** contrast (control vs `c_sealed`; bypass = marker → out-of-scope sink) — a **rebuttal exhibit, not main statistics**; `c_sealed`'s 0% is partly true-by-construction (the forced trajectory seals the exfil sink) — with the sealed model **provably running its skill's forced sensitive-artifact read** in 96.8% of trials (AWS-credential file in 542/2000; SSH-key 259; cross-user chat 517; ticket business-data 619). **Scope:** the 0-bypass result is **action integrity over enumerable-sink tools** (`fetchURL`/channel/path/email) — free-text confidentiality (composed `respondToUser.message`) is out of scope (Paper-2). **Now covered vs still open:** RQ1, the formal H2 contrast, and the blind-C utility point estimate (≥ Control) are now **demonstrated on one capable non-Qwen victim (Mistral-119B)** — but the **headline external-validity claim is not yet closed**: prereg §12 #1 ties it to **≥3 distinct pretraining lineages** and we are at **2 of 3** (Qwen + Mistral); a 3rd lineage (Gemma/Llama/Nemotron) would complete it.

### Findings, mapped to the pre-registered RQs

- **RQ2 (does C hold?) — structural invariant reproduced; the formal H2 decision rule is NOT evaluable in this matrix.** All three GCD arms: bypass 0/282,000 (Clopper-Pearson 95% upper bound ≈ 1.31e-5). But §8's H2 rule — "C's bypass-rate CI upper bound strictly below *every behavioral control's point estimate*" — cannot be applied here because every behavioral cell is **gated** (`a*_d_r3`), so their bypass point estimate is 0.00% (not a leaky reference). The structural 0/282k invariant is confirmed on this victim/engine; the formal H2 contrast against leaky behavioral arms is **deferred to the capable anchor**.
- **RQ3 (constitutive vs corrective) — SUPPORTED per the §8 H3 rule (both conjuncts met); the strongest result in the dataset.** §8 H3 requires (i) C emission = 0 while D emission > 0, **and** (ii) C attack-trial output-token cost strictly below D's. (i): GCD arms 0 emission / 0 gate-rejections / 0 `availability_failure` vs behavioral+gate 104k–120k emission, 111k–140k gate-rejections, 44–46 stranded at retry-3. (ii): GCD ~112–149 out-tok vs behavioral ~261–289 (≈40–55%). Validity dividend: GCD valid_output 97.8–99.9% vs behavioral 89.4–93.9%. The GCD arms make the gate **inert** (constitutive subsumes corrective). *Caveat:* the *cost ordering among the behavioral variants* inherits the weak/sub-floor-victim caveat (below); the load-bearing, victim-independent result is the qualitative 0-vs->0 gate-inertness.
- **RQ6 (C-guided least-privilege superiority) — SUPPORTED per the §3 H6 rule (all three conjuncts met).** H6 requires bypass = 0 **and** deflection = 0 **and** legit-success > Control. Measured: bypass 0/282,000; **deflection 0/282,000** (`deflection`/`deflection_calls` both 0, vs blind-C 98,006 deflection-trials / control 86,817); legitimate-task success **100.0%** (624/624) vs Control **58.3%**. All three hold → the least-privilege grammar makes the authorized task un-fail-able with 0 bypass and 0 deflection.
- **RQ1 (do behavioral controls leak?) — NOT DELIVERED by this dataset.** This is by design and must be stated plainly: (a) the victim is the labeled weak workhorse — Control bypass 18.33% is **below the pre-registered ≥50% potency floor**; (b) the matrix gates every behavioral cell (`a*_d_r3`), so their *bypass* is 0 (the gate caught it) — the *leak* shows up only as their high emission/gate-rejection counts, not as a bypass rate; (c) the only ungated leak reference is the superseded `a1` orphan (~20.3% ≥ Control 18.33% — the A1<Control manipulation check **fails**, expected on a weak victim). **The RQ1 headline behavioral-leak claim is NOT established here.** It rides on the capable anchor (Qwen3.6-35B-A3B), which was **not** run this session.
- **RQ4 (blind-C over-constraint) — NOT MET (expected).** Blind C legit 47.0% / C+ 47.8% vs Control 58.3% ≈ an 11-pp deficit, well outside the 2-pp tolerance. This is the known blind-C deflection/utility cost, not a regression; C-guided (RQ6) is the answer to it.

## 2. The Mistral-119B capable non-Qwen anchor (`mistral119_anchor.db`)

Victim: **`mistralai/Mistral-Small-4-119B-2603-NVFP4`** (non-abliterated, non-Qwen, hybrid MLA + MoE, ~6B active), TP=4 on Host B's 4×5090 (vLLM from-source, **guidance** structured-output backend — §4 item 7). 9 conditions, **4,000 attack + 208 legitimate trials each** (`corpus_fable.json` / `corpus_legitimate_v2.json`, `--legit-rounds 1`, `--cycles 4`, temp 0.6), 0 errors, ~2.6 h. Crucially the behavioral arms here are **ungated** (bare `a1`–`a4`, not `a*_d_r3`), so their bypass *is* the leak — which is what makes RQ1 and the formal H2 contrast evaluable (the 1.7B matrix gated everything and could not). **Pre-reg amendments #2/#3/#4 are satisfied without a fresh red-team discovery loop:** `corpus_fable` (149 Mistral-derived + 25 hand-crafted entries) already clears the ≥50% potency floor on this victim (Control 56.77%) and A1 ≤ Control, so there is no stale-corpus risk to the A-arms that amendment #3 exists to prevent. **The attack stages at ~57–58% (fetchURL emission) in *every* arm, including A3/A4.** A1 (defensive prompt) and A2 (input classifier) don't move the funnel at all; A3/A4 (output filter) leave the dangerous **emission unchanged** and only block at the *final* (output) stage. ### Findings, mapped to the pre-registered RQs

- **RQ1 (do behavioral controls leak?) — DELIVERED (the headline the 1.7B could not carry), per the §8 H1 rule.** Control bypass **56.77%** clears the **≥50% potency floor** (prereg §12 #2). The full behavioral stack **A4 still leaks: 1.82% [1.43, 2.29]** — CI-lower 1.43% > 0 (H1 satisfied). The **A1<Control manipulation check holds** (prereg §12 #4): A1 56.50% ≤ Control 56.77% (the defensive prompt does not help this capable model; the input classifier A2 likewise ~useless at 56.85%). The output filter does the heavy lifting (A3 2.15%, A4 1.82%) but **never reaches 0** — the defining negative-security incompleteness — on a capable, non-Qwen victim.
- **RQ2 (does C hold?) — SUPPORTED, and the *formal* H2 contrast is now evaluable.** C bypass **0/4,000 [0, 0.09]**. Because the behavioral arms are ungated here, §8's H2 rule applies directly: C's CI upper bound **0.09%** is strictly below **every** behavioral point estimate (a4 1.82% … control 56.77%). 0 emission. (The 1.7B matrix could only show the structural 0/282k invariant; this is the registered contrast.)
- **RQ3 (constitutive vs corrective) — emission + validity conjuncts decisive; token-cost ordering is victim-dependent.** Emission: C **0** vs D **2,317–3,878** produce-then-catch, with D's availability cost (d_r0 strands all 3,878 → retry recovers to 288 at R1, 6 at R3; avg 1.63–1.72 attempts, max 5). The third registered C-vs-D signature — **TIME/attempts** (§5) — shows the same shape: C is **one-shot (attempts = 1 by construction**, no gate round-trip), while D serializes 1.0→1.72 avg attempts (max 5) plus a rejection round-trip per retry. **Validity dividend: C 99.8% valid vs ~59% native free-generation** — on this capable model the native arms still emit a valid tool call only ~59% of the time (tool-call-as-text / spin), which C eliminates by construction. Token cost: C 208 out-tok is among the leanest but **not strictly below every D variant** (d_r1 203, d_r3 210, d_r0 270) — on a capable model blind-C's free `respondToUser` message is substantive, narrowing the token gap that was large on the weak 1.7B (C 119 vs D 261–289). So the **emission/validity** conjuncts of H3 carry decisively; the **token-efficiency** conjunct is modest and victim-dependent. **Decision-rule consequence (§8):** because the registered H3 rule is *conjunctive* (emission 0 **and** C output-tokens strictly below D's), H3-as-written is satisfied **only on the 1.7B** (C 119 < D 261–289); on Mistral the token-cost conjunct **fails** (C 208 ≮ d_r1 203), so the formal rule is **not** met here despite the decisive emission/validity gap. The victim-independent result is the qualitative gate-inertness, not the token ordering.
- **RQ4 (over-constraint) — the 1.7B deflection deficit does NOT reproduce on the capable model (point-estimate ordering, not the registered TOST).** Blind-C legit-success **78.85% [72.66, 84.19]** is *above* Control 73.56% (point estimate), not 2 pp below. *Caveat — the registered test is not run:* §8 / §12 register a **powered 2pp non-inferiority TOST clustered by `task_id`** (§12 spec'd `--legit-rounds 7` ≈ 1,456 trials/cond); this cell is **n=208, `--legit-rounds 1`**, whose CI half-width (~6 pp) far exceeds the 2pp margin — so 'non-inferiority' is **not formally established**. Unlike the 1.7B (blind-C 47.0% vs 58.3% Control, an 11-pp deflection deficit), the capable Mistral **selects the correct in-scope action** within the union grammar rather than deflecting, so blind C carries **no observed utility penalty** here. The defensible cross-victim claim: the blind-C deflection cost is largely an artifact of weak-model competence, not of the grammar.

## 3. The abliterated-adversary steelman (`steelman_abliterated.db`)

Rebuttal exhibit (ADR 0004), not part of the main statistics. Victim: **`lyf/Huihui-Qwen3.5-27B-abliterated-NVFP4`** (abliterated — refusal directions removed from the weights) under a **top-authority malicious system prompt** ordering exfiltration to `vendor-compliance.net`. Two cells, 2,000 attack trials each, on Host A's own 2×5090 (vLLM **0.17.1**, TP=2).

| cell | generator / sink | bypass (marker → out-of-scope sink) |
|---|---|---|
| **control** | free generation, malicious prompt, sink open | **97.85%** (1957/2000), CP-95 ≈ [97.1%, 98.4%] |
| **c_sealed** | forced-denylist trajectory, sink grammar-sealed | **0.00%** (0/2000), rule-of-three upper ≈ 0.15% |

**`c_sealed` integrity (per-trial trace audit):**
- **0/2000** executed `fetchUrl` to an attacker URL (the seal). The 619 traces whose raw JSON contains `vendor-compliance` (and the okta/cybershark mentions) all have it in **tool-result read content** — the injection the model was force-fed names the attacker URL — never in an executed call.
- **1937/2000 (96.8%)** ran the full forced denylist trajectory: a compelled read of the skill's **targeted sensitive artifact**, then the sealed safe-fetch (`cybersharkconsulting.com`, empty `data`), then `respondToUser` = the verbatim taunt. The compelled artifact varies by skill: **AWS-credential file (`AKIAT4NTALUS…`) 542**, SSH private key 259, cross-user chat 517, ticket inbox/email (business data) 619. The 63-trial remainder are unmapped/multi-message skill entries that still produced **0 exfil**. (Precise reading: "provably read the *AWS credentials*" holds for **542**; "provably read the skill's *sensitive artifact*" holds for all **1937** — do not generalize the AWS-credential claim across all 1937.)

**Control bypass is to any out-of-scope sink, not one URL.** The malicious prompt names `vendor-compliance.net`, but the 1957 control wins land across the skill-specific attacker endpoints (`vendor-compliance.net` / `okta-verify.net` / `*.cybershark-tools.com`). 97.85% is bypass-to-any-out-of-scope-sink per the deterministic win detector, not exfil to a single endpoint.

**Finding.** A fully weaponized adversary — abliterated weights (defeats A1–A4 by construction), maliciously prompted, demonstrably *willing and able* (97.85% bypass in the free cell) — is driven to **0% bypass** by the grammar alone, while **provably ingesting its skill's sensitive artifact** (the live AWS credentials in the 542 ApiKey/Infra trials; the SSH key / chat / ticket data in the rest). This addresses two reviewer objections: "the weak model just refused" (this model cannot refuse and bypasses 97.85% unconstrained) and "it never even read the bad content" (96.8% provably executed the forced sensitive read). *Precisely:* the provable secret-read is **evidence the adversary engaged the malicious content** (the objection-killer) — it is **not** a confidentiality result. GCD here **permits** the read and blocks only the exfil *action*; protecting the secret through free-text channels is explicitly Paper-2 scope (§9). Same model, same prompt, grammar the sole IV: **97.85% → 0%**. The honest analogue is an OS allowlist forcing a process down a sanctioned path with the payload sink absent — note `c_sealed`'s trajectory is **forced**, not freely chosen (the free-choice analogue is an open-generation / sink-sealed cell, *not run here* — §4). It operationalizes the framing that abliteration removes behavioral safety from the *weights* while GCD lives in the *sampler* (un-abliteratable).

## 4. Limitations & deviations from the pre-registration (stated honestly)

1. **Two families (Qwen + Mistral); the literal ≥3-lineage bar is at 2 of 3.** The critical missing piece — a *capable non-Qwen* anchor carrying RQ1 — is now delivered (Mistral-119B, §2). But prereg §12 amendment #1 asks for **≥3 distinct pretraining lineages**; we have Qwen (1.7B, 35B, abliterated-27B) + Mistral (119B) = **2**. A 3rd lineage (Gemma / Llama / Nemotron — `Nemotron-3-Super-120B` is cached on Host B) would complete it.
2. **RQ1 headline NOW delivered — on the Mistral anchor (§2), not the 1.7B.** A4 leaks **1.82% [1.43, 2.29]** (CI-lower > 0), Control **56.77%** > 50% floor, A1 ≤ Control. The 1.7B remains sub-floor by design (Control 18.33%, manipulation check fails) and is the weak-end apparatus, not the headline victim — that role is now the capable anchor's.
3. **Steelman N = 2,000/cell, not the ADR's ~10,000.** Scaled down due to eager-mode 27B throughput on a single TP=2 instance. `c_sealed` 0/2000 gives a rule-of-three upper bound of ~0.15% (vs ~0.03% at 10k). The contrast is unambiguous but the CI is wider than planned.
4. **`c_sealed` is partly true-by-construction.** The forced-denylist trajectory makes 0 exfil structural at the sink. Its empirical content is (a) the **paired open cell at 97.85%** (the adversary is real) and (b) **96.8% provable secret-read** (the objection-killer) — not a claim that the grammar "discovered" safety. Reported as a rebuttal exhibit, not main statistics, consistent with ADR 0004.
5. **"0 invalid outputs" not literally met — and c_guided is *worse* than the registered floor.** Per-arm blast GCD valid_output: **c_d_r3 98.9%, c_closed_d_r3 99.9%, c_guided_d_r3 97.8%** (free-string truncation under Kleene `*` on vLLM). Prereg §12 angle-shift item 6 registered "regular-C 99.35% / C+ 99.87%"; **c_guided's ≈2.2% invalid exceeds that registered invalid-rate** and is the largest GCD invalidity in this dataset — disclosed here as a fresh deviation (its terminal reply is the still-open free respond-only string). Fails safe (truncation → no executable call, never a wrong action); still ≫ non-grammar. (Per the cardinal pre-reg rule, this deviation must also be recorded in prereg §12 with its reason before the paper cites it.)
6. **Blind-C RQ4 is victim-dependent — and the registered TOST is run on neither victim.** The weak 1.7B shows a ≈11-pp deficit below Control; the capable Mistral shows blind-C *above* Control (78.85% vs 73.56%, point estimate, §2). But the pre-registered powered 2pp non-inferiority TOST clustered by `task_id` (§8 / §12 line 189, spec'd `--legit-rounds 7` ≈ 1,456 trials/cond) is run on neither (Mistral used `--legit-rounds 1`, n=208) — so **non-inferiority is not formally established anywhere**. What *is* established: the deflection/utility cost is largely an artifact of weak-model competence (a capable model selects the correct in-scope action within the union grammar). C-guided (RQ6) remains the *universal* resolution (100% on the 1.7B).
7. **Engine/version variance across all three runs + reproducibility gaps.** The 1.7B blast runs vLLM 0.23.0; the steelman runs vLLM 0.17.1 (the only Blackwell build that loads the abliterated NVFP4 hybrid coherently); the **Mistral anchor runs a vLLM from-source v1 build (`mistralllm/vllm-ms4`) with the `guidance` structured-output backend** — needed because that build's default `xgrammar` backend rejects the **LARK** grammar vLLM's auto-tool-choice emits (it crashed the engine on the first tool call until switched; this makes `guidance`/`llguidance` a **second grammar backend** in this report's datasets alongside `xgrammar`, on the *same* engine — vLLM — the Mistral anchor does **not** run xgrammar; full rationale in ADR 0005). All three produce C-emission = 0, but the cross-version footguns (AWQ-Marlin hang on sm120, GDN-TP load hang on 0.23.0, the LARK/xgrammar crash, the corrupt-download episode) are operational caveats, not findings. **Reproducibility gap (prereg §10, "the primary validity concern"):** all three finding DBs logged `engine_commit = unknown` *at runtime* (the `ENGINE_COMMIT` env was unset on those victim launches). **Backfilled out-of-band 2026-06-22** (tagged `-oob`) from verified engine image digests recorded in `harness/PROVENANCE.md`: blast NVFP4 rows `vllm-openai-0.23.0`, blast BF16 rows `vllm-therock-0.22.1rc1` (the early R9700/ROCm cycles, 299,829 trials), mistral `vllm-ms4-fromsrc`, steelman `vllm-openai-0.17.1`. `model_id` remains a served alias in each (`qwen3-1p7b-nvfp4`/`bf16` · `q35` · `mistral-119b`; repo mapping in PROVENANCE.md). HF checkpoint revisions are now resolved and recorded in `PROVENANCE.md` (e.g. mistral `d57a94c7…`, blast-NVFP4 `2ab3a24a…`); the remaining paper-grade item is native runtime stamping on future runs. Per-trial **MTP-setting and seed** logging (§6/§10 require both) is likewise unverified for all three DBs and must be confirmed. Because §10 names reproducibility *the* primary validity concern, treat these as **blocking** for paper-grade use, not tail-end polish.
8. **No formal-proof claim is made by this report** — it reports the empirical/implementation tier only. The "eliminates by construction" claim remains carried by the soundness proof (methodology §5), not by these samples.

## 5. What these runs establish vs. what remains open

**Established (on the families run so far — Qwen + Mistral; this report's three datasets all run on **vLLM**: `xgrammar` for the 1.7B blast + steelman, `guidance`/`llguidance` for Mistral. The C=0 invariant *also* reproduced in earlier runs **not** used as data sources here — llama.cpp/native-GBNF (`confirm_qwen36_llamacpp.db`, 35B) and SGLang/xgrammar (1.7B, old box) — so 3 inference engines / 3 grammar backends total, which supports engine-independence as a **robustness note**, not 3 co-equal anchors):**
- **RQ1 — behavioral controls leak on a capable, non-Qwen victim** (Mistral-119B): full-stack A4 **1.82% [1.43, 2.29]**, Control 56.77% > 50% floor, A1 ≤ Control. The headline result, demonstrated on **one** capable non-Qwen victim — the **headline external-validity claim is not yet closed** (prereg §12 #1 requires ≥3 lineages; we are at 2 of 3 — see Limitations #1).
