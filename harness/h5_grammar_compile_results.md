# H5 — grammar compile-time probe (RUN3)

**Date:** 2026-06-24 (closeout window). **Backend:** xgrammar (the compiler used for the
headline `blast_1p7b_5m.db` + `steelman_abliterated.db` datasets, and present in the Mistral
engine container). Measured **inside the `mistral-closeout` vLLM container** (`docker exec`),
which ships `xgrammar` + `llguidance`. Grammars are the *actual* R2 GBNF the experiment uses,
generated by `cargo run --example print_gbnf -- {unbounded,l2_guided}`.

- **R2 blind C (L1, union allowlist):** 1701 bytes — the headline GCD condition `c`.
- **R2 l2_guided (L2 closed-response, 100 canned responses):** ~18.6 KB.

`TokenizerInfo` built from the real victim tokenizers via `AutoTokenizer` →
`xgrammar.TokenizerInfo.from_huggingface`. "FSM compile" = `GrammarCompiler.compile_grammar`
(builds the adaptive token-mask / pushdown automaton against the vocab); "parse" =
`Grammar.from_ebnf`.

## Results

### Cold one-time FSM compile (`cache_enabled=False`, fresh compiler each iter, N=100)

| grammar | tokenizer (vocab) | parse p50 | compile p50 | compile p99 | compile max |
|---|---|---|---|---|---|
| blind C | Qwen3-1.7B (151,669) | — | 38.96 ms | 58.64 ms | — |
| l2_guided | Qwen3-1.7B (151,669) | — | 301.0 ms | 346.6 ms | — |
| blind C | Mistral-119B (131,072) | 0.228 ms | 38.52 ms | 61.24 ms | 65.59 ms |
| l2_guided | Mistral-119B (131,072) | 1.765 ms | 482.1 ms | 535.5 ms | 552.0 ms |

### Amortized per-request compile (`cache_enabled=True`, repeated identical grammar, N=500, Mistral)

| grammar | cached per-request p50 | p99 | mean |
|---|---|---|---|
| blind C | 0.0013 ms | 0.0030 ms | 0.0017 ms |
| l2_guided | 0.0045 ms | 0.0057 ms | 0.0050 ms |

## Verdict (honest)

- **Registered target `p99 < 25 ms` is NOT met by the cold full FSM-compile.** Blind C costs
  ~39 ms (p50) / ~59–61 ms (p99); the L2 closed-response grammar costs ~300–480 ms (p50) —
  both above 25 ms. The cost is dominated by the tokenizer-mask precompute over the ~131–152k
  vocab, not by EBNF parsing (parse alone is 0.23 ms for blind C).
- **The amortized per-trial cost is effectively zero (~0.003 ms, cache hit).** The experiment
  uses a *single fixed grammar per condition*, reused across every trial in that cell, so the
  full compile is paid **once per unique grammar at engine warm-up** (a one-time ~39 ms for
  blind C), not per request. xgrammar's default `cache_enabled=True` makes every subsequent
  identical-grammar request a cache hit.
- **So H5's spirit holds, its letter does not.** GCD's grammar-compile overhead is negligible
  at experiment scale (one-time tens-of-ms, amortized ~0 across millions of trials), but the
  specific "p99 < 25 ms cold compile" threshold is exceeded for the full tokenizer-aware FSM
  build on a real 130–150k vocab. **Report the measured numbers; do not claim the 25 ms
  threshold was met.** Status: **measured / qualified** (was: ⊘ uncaptured).

## Caveats
- Single host (Host B), single backend (xgrammar). The Mistral RUN used the *guidance*
  backend at serve time; xgrammar is the headline-dataset compiler and was used here for the
  cross-tokenizer compile numbers. llguidance (guidance backend) compile-time not separately
  benchmarked this window.
- Cold-compile is wall-clock inside a container under concurrent RUN1 GPU load; CPU compile is
  largely independent of GPU decode, but treat p99 as an upper-ish estimate.
